## Frontend Overview

This is the VitePress + Vue 3 static site frontend for the CourseSharing platform.

**Key Technologies:**
- VitePress 1.6.4 - Static site generator
- Vue 3.5.30 - UI framework
- Custom FileTree component for folder navigation

## Common Commands

| Command | Description |
|---------|-------------|
| `npm run dev` | Generate structure and start VitePress dev server (debug mode) |
| `npm run build` | Build production static site |
| `npm run generate` | Only generate course pages from manifest |
| `npm run deploy` | Generate and build for production |

**Note:** Debug mode uses `docs-debug/` directory, production uses `docs/`.

## Key Files

### Configuration
- **`docs/.vitepress/config.js`**: VitePress config with navigation, sidebar generation from `course-info.json`

### Components
- **`docs/.vitepress/theme/components/FileTree.vue`**: Recursive file tree component with:
  - Folder sorting (folders before files, alphabetical)
  - Recursive collapse/expand
  - Streaming downloads with progress
  - ZIP folder downloads

### Build Script
- **`scripts/generate-structure.js`**: Generates course pages from `manifest.json` and `course-info.json`

### Data Files (Auto-generated)
- **`docs/.vitepress/public/manifest.json`**: Global file manifest mapping paths to R2 hashes
- **`docs/.vitepress/public/course-info.json`**: Course metadata (titles, categories, descriptions, ranks)

**Do NOT edit these data files directly** - they are generated by the update scripts.

## Course Categories

Courses are organized into:
- `MR`: Major Require
- `ME`: Major Elective
- `FE`: Free Elective
- `GE`: General Education
- `UC`: University Core
- `Other`: Other Materials

## Important Technical Notes

1. **Folder Sorting**: Folders always appear before files, sorted alphabetically at all levels
2. **Case Sensitivity**: Category names use normalization - "Other" is converted to "other" internally
3. **Worker URLs**: `generate-structure.js` uses:
   - Debug mode: `http://localhost:8787`
   - Production: `https://course-sharing-download.fm-course.workers.dev`
4. **Page Generation**: Course pages in `docs/courses/` are auto-generated from templates - do not edit directly

## Content Updates

To update course content or metadata:
1. Modify files in the root `content/` directory
2. Run the update script from the root directory:
   ```bash
   cd ~/CourseSharing
   bash update/scripts/update-all.sh
   ```

This will regenerate `manifest.json`, `course-info.json`, and the course pages.
